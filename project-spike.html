Spike Originator Model

The Challenge

Legacy fraud detection flagged too many false positives, overwhelming investigators and missing claim-level precision. The goal was to reduce noise and surface the riskiest claims for review.

System Architecture

Data Pipeline

Raw claims data flows through a PySpark-based ETL pipeline, performing feature extraction across 40+ behavioral signals including billing frequency, procedure code distributions, and provider-to-patient ratios. Data is partitioned by submission date and indexed for efficient retrieval.

Hybrid Detection Framework

Stage 1 — Unsupervised Outlier Detection: Isolation Forest and DBSCAN identify anomalous claim patterns at the provider level using Mahalanobis distance metrics. Outlier scores are normalized using Z-score standardization.

Stage 2 — Supervised Classification: LightGBM classifier trained on 500K+ labeled claims refines predictions. Features include temporal patterns, dollar thresholds, and historical provider behavior vectors. Hyperparameters optimized via Optuna with 5-fold stratified cross-validation.

Stage 3 — Score Aggregation: Ensemble scoring combines unsupervised outlier probabilities with supervised class probabilities using weighted averaging. Final risk scores ranked and filtered by configurable dollar thresholds.

MLOps & Monitoring

Models registered in MLflow with versioning and experiment tracking. Airflow DAGs orchestrate quarterly retraining cycles with automated data validation, model evaluation against holdout sets, and deployment gates. Tableau dashboards track precision/recall trends, claim volumes, and estimated savings.

Impact

  • 11% increase in precision while maintaining recall above 65%
